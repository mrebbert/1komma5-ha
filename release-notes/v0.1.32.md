## What's New

### New Services

- **`onekommafive.get_cheapest_window`** — find the cheapest contiguous N-minute window in the price forecast. Returns start/end timestamps and average price for use in automations (dishwasher, washing machine, EV, heat pump scheduling).
- **`onekommafive.get_most_expensive_window`** — mirror service for load-shedding automations.

Both accept optional `earliest_start` / `latest_end` constraints and a `config_entry_id` for multi-system accounts.

### Credential lifecycle

- **Re-authentication flow** — when 1KOMMA5° credentials become invalid, Home Assistant automatically shows a "Re-authentication required" notification. Enter new credentials in place without losing sensor history.
- **Reconfigure flow** — proactive credential updates via Settings → Devices & Services → 1KOMMA5° → Reconfigure.

### New Sensors

- **Negative Price Slots Tomorrow** sensor — count of 15-min slots tomorrow with negative price (available after ~13:00 CET).
- **Cheapest Hour Now** binary sensor — ON when the current 15-min slot is the cheapest in the next ~30h of forecast.

### Improvements

- **Cheap Electricity** binary sensor now updates dynamically every 15 minutes (was previously only updated on coordinator refresh).
- **Optimization aggregation sensors** (`optimization_total_cost`, `optimization_energy_bought`, `optimization_energy_sold`) no longer declare `state_class: total`. They are daily snapshots that reset at midnight; recording them as totals without a `last_reset` polluted Long-Term Statistics with a midnight drop.

### Under the hood

- **37 unit tests** covering all pure helper functions and translation file consistency.
- **GitHub Actions** runs the test suite on every push and pull request.
- **GitHub issue templates** for bug reports and feature requests.
- Major refactoring: extracted `helpers.py`, introduced `OneKomma5BaseCoordinator[T]`, `OneKomma5AccumulatingSensor`, `QuarterHourUpdateMixin`, `system_device_info` helpers; split `sensor.py` into `sensor.py` / `sensor_descriptions.py` / `sensor_entities.py`; consolidated reauth + reconfigure flows; converted EV number entities to a description-based pattern.

## Breaking changes

- **Minimum Home Assistant version is now 2024.10** (required for `_get_reauth_entry`, `_get_reconfigure_entry` and `data_updates` helper). Users on older HA versions need to upgrade before installing this release.
